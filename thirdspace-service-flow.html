Entry Points
4. All Entry Points Into the Reporting Flow
Multiple ways to start a report — SMS is primary, but QR and admin-initiated magic links are also supported.
📱
SMS to Thirdspace number
Primary entry point. Staff texts any message to the dedicated number. System replies with magic link. Works without wifi, without an app, at 2 AM in a loud bar. Number displayed on staff cards, behind the bar, and in break rooms.
▦
QR code scan
Secondary entry point. QR code posted behind the bar or on the wall opens the report wizard directly in the browser. Same wizard, same flow — just a different entry point. Useful for venues with good wifi. QR codes are venue-specific (pre-linked to venue ID).
🔗
Admin-sent magic link
Follow-up entry point. GM creates an incident in the dashboard and sends magic links to specific staff. Used when the GM learns about an incident after the fact and needs individual reports from staff who were present. Also used for witness statements.
📄
OCR from paper report
Bridge for existing workflows. Staff or admin takes a photo of a paper incident board/report. System OCRs it and pre-fills the wizard fields. Staff reviews, corrects, and submits. Helps venues transition from paper to digital without changing habits overnight.
Data Architecture
5. How Data Flows Through the System
From individual staff reports to a complete case file ready for TPA review.
1
Individual Incident Reports
Each staff member submits their own account. Multiple reports can be attached to the same incident. Each report = one person's perspective.
2
Investigative File
System aggregates all reports + evidence + witness info into a single investigative file. Completeness is tracked against the minimum information set for that incident type (e.g., Slip & Fall needs: patron info, staff statements, witness statements, video, maintenance records, photos).
3
Admin Review & Enrichment
GM reviews the file, adds context, sends additional magic links for missing statements, uploads supplementary evidence (POS data, cleaning logs, security agreements). Completeness score updates in real-time.
4
Case File → TPA Handoff
Complete investigative file is packaged as a structured case file. Includes all reports, evidence, defensibility assessment, and timeline. TPA can review and action on it — this is what prevents cases from going to trial.
Open design questions
  • Should the Thirdspace SMS number be per-venue or shared across all venues? Per-venue simplifies context detection but adds operational complexity.
  • How do we handle multiple incidents on the same night? After the magic link is used, does texting again create a new incident or link to the existing one?
  • Should the QR code open the full wizard or a simplified "quick capture" (just photos + type) with a follow-up link for details?
  • Camera footage reminder cadence (7/14/25 days) — should this be configurable per venue based on their DVR retention settings?
  • Do we need a "nothing happened" shift report that confirms no incidents? Helps with the defensibility narrative (absence of reports = good operations, not missing data).
  • OCR feature scope for v1: full form pre-fill or just text extraction that the user then manually places into fields?
